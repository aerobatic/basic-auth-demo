HTTP Basic Auth Demo

We now recommend you use the password-protect plugin instead of basic-auth.

Sometimes you just need to provide a global basic username/password protection on an app. Often HTTP basic authentication is a “good enough” solution. This plugin lets you secure all or part of your app with a username and password stored in environment variables.

We offer two flavors of basic auth: standard and custom. With standard basic auth, unauthenticated users are prompted to enter the website username and password in the browser's built-in authorziation dialog. This gets the job done, but it doesn't allow for any customization or branding. Custom basic auth, available only for paid plans, allows you to present your website visitors a fully custom login html page.
